# Task 24 — Replace `docker run` `-v/--volume` Usage With `--mount`

## Goal

Convert every place where `aicage` builds or documents `docker run` volume flags (`-v`, `--volume`, any `--volumes`
alias you may have) so the produced `docker run` command uses only `--mount ...` for bind mounts.

This is a mechanical refactor plus tests.

## Scope

- Code: any module that constructs `docker run` CLI arguments (including helpers/utilities).
- Docs/examples/tests: any embedded snippets that show `docker run -v ...`.

## Definitions

- Bind mount: host path ↔ container path mapping.
- Named volumes / anonymous volumes:
  - If `aicage` supports them today, decide whether to:
    - keep them as named volumes but still express via `--mount type=volume,src=...,dst=...`, or
    - leave as-is (only if you find no usage, or explicitly out-of-scope).

Prefer expressing both bind + named volumes via `--mount` for consistency.

## Implementation steps

1. Inventory current usage
   - Search the repo for:
     - `docker run`
     - `-v`
     - `--volume`
     - `--volumes`
     - `:ro` / `:rw` patterns
     - any internal builder names like `volume_args`, `mount_args`, `binds`
   - List call-sites (file + function) where volume flags are produced.
2. Create a mount builder (single source of truth)
   - Introduce / reuse a helper that produces mount args:
     - Input: mount objects like:
       - `{type: "bind", src, dst, readonly?: bool, ...}`
       - `{type: "volume", src, dst, readonly?: bool, ...}`
     - Output: list of docker args:
       - `["--mount", "type=bind,src=...,dst=...,readonly"]` (readonly only if needed)
   - Ensure:
     - stable ordering of kv-pairs (helps tests)
     - correct shell escaping by passing as argv elements (don’t pre-quote)
3. Convert each call-site
   - Replace `-v host:dst[:mode]` creation with `--mount type=bind,src=host,dst=dst[,readonly]`.
   - Replace `--volume name:dst` creation with `--mount type=volume,src=name,dst=dst`.
   - If you previously used `:rw`, omit it (rw is default); keep ro as `readonly`.
4. Update docs/tests/examples
   - Replace any `docker run -v ...` snippet with the `--mount` form.
   - Keep examples consistent with how `aicage` prints debug output.
5. Tests
   - Add/adjust unit tests that assert the produced argv contains:
     - `--mount` entries only
     - no `-v`, `--volume`, `--volumes`
   - Add a couple of representative cases:
     - rw bind mount
     - ro bind mount
     - named volume (if supported)
     - multiple mounts ordering (stable)
6. Backward compatibility (optional but recommended)
   - If `aicage` itself has CLI options named `--volume` (not docker passthrough), keep parsing them but emit
     `--mount`.
   - If users can pass raw docker args through (including `-v`), do not rewrite user passthrough; only refactor
     what `aicage` itself generates.

## Acceptance criteria

- `aicage`-generated `docker run` commands do not contain `-v`, `--volume`, or `--volumes`.
- All mounts generated by `aicage` are expressed with `--mount ...`.
- Existing behavior remains the same (same paths, same ro/rw semantics).
- Tests updated/added and passing.

## Notes

Prefer `--mount` because it’s explicit (type=bind vs volume), extensible, and easier to validate in code.
